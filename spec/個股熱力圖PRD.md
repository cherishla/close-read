# 個股熱力圖 - 產品需求文件 (PRD)

> **最後更新**: 2026-02-03
> **功能定位**: 個股層級的篩選與視覺化工具
> **市場範圍**: 美股市場

---

## 產品概述

### 產品目標

提供一個**指標驅動的個股篩選器**，讓使用者能快速在特定族群（如 S&P 500）內找到表現突出的個股，並以熱力圖方式直觀呈現。

核心問題：
- 在 S&P 500 內，哪些個股表現最好？
- 用什麼指標來評判（漲跌幅、EPS 成長、Above MA 等）？
- 如何快速視覺化比較數百檔股票？

### 核心價值

1. **直接看個股**：不需經過 Sector/Industry 層級，直接篩選個股
2. **多指標支援**：支援價格、基本面、評價、風險等多維度指標
3. **視覺化比較**：熱力圖模式快速識別強弱股
4. **靈活範圍**：支援多種族群範圍（S&P 500、S&P 400、NASDAQ 等）

### 產品定位

- **個股篩選器（Stock Screener）**：指標驅動的選股工具
- **補足產品線缺口**：市場總覽 → **個股篩選器** → 個股儀表板
- **與產業熱力圖互補**：
  - 產業熱力圖 = 產業驅動選股（Sector → Industry → Stock）
  - 個股熱力圖 = 指標驅動選股（直接篩選 Stock）

## 功能範圍

### MVP 階段功能

#### 1. 族群範圍選擇

支援以下族群：
- **S&P 500**：大型股
- **S&P 400**：中型股
- **NASDAQ**：科技股為主

**與大盤市場多空整合**：
- 從「大盤市場多空」的族群表現區塊跳轉時，自動過濾到對應族群

#### 2. 指標選擇

**價格表現類**：
- 區間漲跌幅（1D / 5D / 1M / 3M / YTD）
- 相對報酬（vs S&P 500）
- 52 週區間位階

**基本面動能類**：
- TTM 營收年增率
- TTM EPS 成長率
- 營業利益率變化

**風險/市場情緒類**：
- Above 50D MA（是/否）
- Above 200D MA（是/否）
- 成交金額相對強度（Turnover Ratio）

**評價水位類**：
- PE 位階
- PB 位階
- 相對同業 PE 溢價/折價

#### 3. 熱力圖呈現

**格子顏色**：
- 依所選指標的數值決定
- 顏色設計與產業熱力圖一致（沿用現有配色邏輯）

**格子大小**（使用者可切換）：
- **選項 1：依市值**（預設）
  - 格子大小 = 個股市值佔所選族群總市值的比例
  - 與產業熱力圖邏輯一致，反映市場結構
- **選項 2：依區間成交量**
  - 格子大小 = 個股在指定區間（如 1D/5D/1M）的成交量佔比
  - 反映交易活躍度
- **選項 3：依區間成交值（Turnover）**
  - 格子大小 = 個股在指定區間的成交金額（Price × Volume）佔比
  - 反映資金流動量
- **選項 4：一樣大小**
  - 所有格子等大，方便視覺比較
  - 每檔個股獲得同等視覺重量

**格子內容**：
- 股票代號（Ticker）
- 指標數值（hover 顯示更多資訊）

#### 4. 分組顯示

**使用者可切換是否依 Sector 分組顯示**：

- **不分組**（預設）：
  - 所有個股平鋪顯示
  - Tree map 演算法自動最佳化排列
  - 適合純指標驅動的篩選

- **依 Sector 分組**：
  - 個股依所屬 Sector 分組顯示
  - 每個 Sector 為一個區塊，內部顯示該 Sector 的個股
  - 保留 Sector 的視覺邊界（類似產業熱力圖的階層結構）
  - 適合同時關注產業結構與個股表現

**參考設計**：
- TradingView Stock Heatmap 的 "Group by Sector" 功能
- 現有產品的類似功能

#### 5. 基本互動

- **點擊個股**：跳轉至個股儀表板
- **Hover 顯示**：Tooltip 顯示詳細資訊（股票名稱、當前價格、指標數值等）
- **族群切換**：下拉選單切換族群範圍
- **指標切換**：下拉選單或按鈕切換顯示指標
- **格子大小切換**：下拉選單切換格子大小計算方式
- **分組顯示切換**：Toggle 或 Checkbox 切換是否依 Sector 分組

### Phase 2 功能（未來擴充）

1. **更多族群範圍**：
   - Mega-cap（Top 50）

2. **多條件篩選**：

5. **儲存篩選條件**：
   - 使用者可儲存常用的篩選設定

6. **匯出功能**：

- **依 Sector 分組顯示**：Toggle 或 Checkbox（預設不勾選）

### 熱力圖設計考量

#### 與產業熱力圖的差異

**層級結構**：
- **產業熱力圖**：階層結構（Sector → Industry → Stock）
- **個股熱力圖**：扁平結構（直接顯示所有個股），但可選擇依 Sector 分組

**視覺呈現彈性**：
- 個股熱力圖提供更多視覺化選項（格子大小、分組/不分組）
- 產業熱力圖的格子大小固定為市值權重

#### 格子排列方式

**不分組模式**：
- 使用 Tree map 演算法自動最佳化排列
- 根據選定的「格子大小」計算方式排列
- 所有個股平鋪，無 Sector 邊界

**依 Sector 分組模式**：
- 每個 Sector 形成一個視覺區塊
- Sector 區塊大小 = 該 Sector 內所有個股的格子大小總和
- Sector 內部的個股使用 Tree map 演算法排列
- 保留 Sector 邊界線或標籤，清楚區分不同產業

**參考**：TradingView Stock Heatmap 的 "Group by Sector" 切換功能

#### 顯示數量

**MVP 階段**：
- 全部顯示（S&P 500 = 500 檔，S&P 400 = 400 檔，NASDAQ ≈ 3000 檔）
- NASDAQ 可能需要特殊處理（數量過多）：
  - 選項 1：只顯示前 500 名（依市值）
  - 選項 2：提示使用者「數量過多，建議使用篩選條件」（Phase 2 功能）

**Phase 2**：
- 提供篩選條件，只顯示符合條件的股票
- 提供「Top N」選項

### 互動設計

**點擊行為**：
- 點擊格子 → 跳轉至該股票的個股儀表板

**Hover 效果**：
- 顯示 Tooltip：
  - 股票名稱 + Ticker
  - 當前價格
  - 指標數值（例如：1D 漲幅：+3.5%）
  - 其他關鍵資訊（市值、Sector 等）

**族群/指標切換**：
- 下拉選單切換後，熱力圖即時更新

## 與其他功能的整合

### 從大盤市場多空跳轉

**跳轉路徑設計**：

```
┌─────────────────────────────────────────┐
│ 族群表現          [產業分析 →]         │  ← 跳轉至產業熱力圖
├─────────┬─────┬─────────┬───────────────┤
│ S&P 500 │+1.2%│   65%   │    70%        │  ← 點擊此列 → 個股熱力圖（族群=S&P 500）
│ S&P 400 │+0.8%│   58%   │    62%        │  ← 點擊此列 → 個股熱力圖（族群=S&P 400）
│ NASDAQ  │+1.5%│   68%   │    75%        │  ← 點擊此列 → 個股熱力圖（族群=NASDAQ）
└─────────┴─────┴─────────┴───────────────┘
```

**兩條跳轉路徑**：

1. **產業視角** → 點擊「產業分析 →」按鈕
   - 跳轉至：產業熱力圖
   - 用途：從產業結構角度分析市場

2. **個股視角** → 點擊特定族群列
   - 點擊「S&P 500」列 → 跳轉至個股熱力圖，自動選擇「族群範圍 = S&P 500」
   - 點擊「S&P 400」列 → 跳轉至個股熱力圖，自動選擇「族群範圍 = S&P 400」
   - 點擊「NASDAQ」列 → 跳轉至個股熱力圖，自動選擇「族群範圍 = NASDAQ」
   - 用途：直接查看特定族群內的個股表現

**跳轉參數**：
- 族群範圍：自動帶入（根據點擊的列）
- 指標：預設顯示「1D 漲跌幅」（或根據使用者最後選擇的指標）
- 格子大小：預設「依市值」
- 分組顯示：預設「不分組」

**設計優點**：
- 使用者意圖清晰（看產業 vs 看個股）
- 操作簡單（一次點擊到達目標）
- 參數自動帶入（減少手動選擇）
- 兩個功能都有明確入口

### 與產業熱力圖的關係

**互補，不重疊**：
- **產業熱力圖**：從產業角度看個股（Sector → Industry → Stock）
- **個股熱力圖**：從指標角度看個股（直接篩選 Stock）

**產業熱力圖內的 (C) 單一產業層級仍然保留**：
- 使用者在產業熱力圖內選擇特定 Industry 後，可看到該產業內的個股
- 這是「從產業出發」的使用情境

**個股熱力圖提供的是**：
- 「從指數/族群出發」的使用情境
- 跨產業比較（例如 S&P 500 內所有產業的股票一起比較）

### 與個股儀表板的整合

- 點擊個股熱力圖內的任一格子 → 跳轉至該股票的個股儀表板
- 提供完整的個股深度分析

## 資料需求

### 資料來源

| 資料類型 | 來源 | 狀態 |
|---------|------|------|
| S&P 500 成分股清單 | 成分股資料 | ✓ 已有 |
| S&P 400 成分股清單 | 成分股資料 | ✓ 已有 |
| NASDAQ 成分股清單 | 成分股資料 | ✓ 已有 |
| 個股 daily 行情 | EOD 資料 | ✓ 已有 |
| 個股基本面資料 | 財報資料 | ✓ 已有 |

### 計算邏輯

**所有指標的計算邏輯與產業熱力圖一致**，詳見 [產業熱力圖spec.md](../產業熱力圖spec.md)

**差異點**：
- 產業熱力圖：指標在 Sector/Industry 層級聚合計算
- 個股熱力圖：指標在個股層級直接計算

### 更新頻率

- **MVP 階段**：daily 資料，每日更新一次
- **未來擴充**：可升級為即時資料（基於個股即時行情）

## 技術實現考量

### 與產業熱力圖共用技術

雖然是獨立功能，但可以**共用底層技術**：
- **熱力圖渲染元件**：UI component 可重用
- **指標計算邏輯**：API 層可共用（不同的查詢參數）
- **視覺設計風格**：配色、布局保持一致
- **資料 API**：可能只是不同的 endpoint 或參數

### 效能考量

**挑戰**：
- S&P 500 有 500 檔股票，需要即時渲染數百個格子
- 切換指標時需要快速重新計算和渲染

**優化方向**：
- 前端：虛擬滾動（只渲染可見區域的格子）
- 後端：預先計算常用指標，cache 結果
- 考慮分頁或限制顯示數量（例如 Top 200）
