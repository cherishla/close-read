# 台股盤後決策系統 — 專案目標

> 版本：v2.0 | 更新：2026-05-02
> 本文件定義產品的**使用者、核心問題、成功定義與功能邊界**。
> 技術規格與實作細節請見 `PRD.md`。

---

## 一、使用者是誰

**每天收盤後至隔日開盤前，自主研究台股的個人投資者。**

他/她不仰賴他人推薦，習慣自己看資料後做判斷。目前面對的困境是：
- 現有工具資料很多，但分散在不同功能頁，要花很長時間才能拼湊出完整圖像
- 每天的資訊量太大，不知道應該從哪裡開始看、看到什麼程度可以停
- 看到數字，但不確定「這個數字在歷史上算高還是低？」
- 看完資料後，無法確認自己的判斷邏輯有沒有遺漏什麼

---

## 二、使用者每天想回答的問題

使用者每日研究有五個核心問題，依重要性排序：

### 問題一：今天大盤健不健康？
- 漲跌是廣泛的還是集中的？
- 上漲家數、創新高家數是否支撐指數？
- 成交量是放大還是萎縮？
- 今天有什麼異常值得注意？

### 問題二：資金在往哪裡跑？
- 哪些族群今天資金淨流入最多？
- 法人（外資、投信）在押注哪個方向？
- 資金是集中在少數族群，還是廣泛分散？

### 問題三：這個族群近期走勢是轉強還是轉弱？
- 相對大盤的強弱是持續還是剛開始？
- 法人是持續買超還是開始退出？
- 族群內部的廣度（上漲家數）健不健康？

### 問題四：族群裡哪些個股值得進一步研究？
- 誰在領漲？誰是跟漲？誰已經轉弱？
- 有沒有相對強弱突出、又有法人支撐的股票？

### 問題五：這支股票有沒有足夠的支撐？
- 籌碼面：外資/投信/主力怎麼動？大單是否集中？
- 技術面：近期走勢是否延續？
- 基本面：評價是否合理？盈餘成長是否支撐？

## 三、「研究完成」的定義

使用者一次完整的盤後研究結束後，應該能清楚回答：

| 層次 | 問題 | 對應功能 | 目前狀態 |
| ---- | ---- | -------- | -------- |
| 大盤 | 今天市場擴散還是集中？ | BlockA 廣度指標 + narrative | ✅ 已有 |
| 資金 | 哪個族群資金最強？法人怎麼看？ | BlockC + BlockD 熱力圖 | ✅ 已有 |
| 族群 | 這族群近期是轉強還是持續強？ | SectorStats RS 折線 + 買賣超 | ✅ 已有 |
| 個股 | 這支股票在族群中的相對位置如何？ | BlockE 三 Tab | ✅ 已有 |
| 個股 | 這支股票籌碼面有支撐嗎？ | StockPage 籌碼 | ✅ 已有 |
| 個股 | 這支股票基本面/評價合理嗎？ | 個股基本面概覽 | ⚠️ 待建立 |

## 四、V1 產品原則（不做系統交易判斷）

這個工具定位為**狀態呈現工具，不是交易決策系統**。

V1 明確不做的事：

- 不提供買進 / 賣出建議
- 不輸出明確多空結論作為交易訊號
- 不做個股評分或排名推薦
- 不預測股價走向

V1 提供的是：

- 客觀呈現市場狀態（廣度、資金、強弱）
- 提供每個數字的歷史位置感（百分位）
- 點出異常值讓使用者自己判斷
- AI 評語描述狀態，不給方向

**將判讀權保留給使用者，是這個工具最重要的設計原則。**

## 五、研究漏斗（User Flow）

研究路徑是 Top-down，但使用者可以從任何一層跳入，不被強制線性閱讀。

```
主頁（Market Overview）
  ├── 看大盤廣度、指數趨勢       → 回答問題一：今天市場健不健康？
  ├── 看資金流入/流出排行         → 回答問題二：資金往哪裡跑？
  ├── 看族群熱力圖（多指標）      → 鎖定今日強弱族群
  └── 點族群格子
         ↓
SectorPage（族群頁）
  ├── 看 RS 折線 + 買賣超走勢    → 回答問題三：族群轉強還是轉弱？
  ├── 看廣度、三大法人分解        → 確認族群內部結構
  ├── 看量價合一圖                → 確認成交量是否配合走勢
  ├── 看個股結構三 Tab            → 回答問題四：哪些個股值得研究？
  └── 點個股列 → 右側 StockDetailPanel 即時更新
         ↓（若需要深入研究）
StockDetailPanel（族群頁右欄）
  ├── 快速確認族群排名、相對強弱  → 判斷個股在族群中的位置
  ├── 看籌碼摘要（外資/投信等）   → 初步確認資金結構
  └── 點「完整分析 →」
         ↓
StockPage（個股頁）
  ├── 看近 20 日 K 線 + 量        → 確認走勢與量能
  ├── 看法人籌碼細節               → 回答問題五（籌碼面）
  ├── 看基本面概覽（P/E、P/B 等） → 回答問題五（基本面）
  └── 看相關新聞                   → 補充事件背景
```

## 七、兩段式使用體驗

產品需要同時服務**快速掃描**和**深入研究**兩種使用模式：

### 第一段：快速掃描（3–5 分鐘）

> 「今天市場發生了什麼？我應該關注哪個方向？」

流程：大盤健康度確認 → 資金流向定位 → 最強族群識別 → 族群內領漲股鎖定

設計原則：每一層資訊都要能在 30 秒內得出結論，不需要翻到其他頁面。

### 第二段：深入研究（不限時）

> 「這支股票值不值得進場？我的判斷邏輯有沒有遺漏？」

流程：個股籌碼細節 → 區間走勢確認 → 基本面/評價核對 → AI 輔助交叉驗證

設計原則：每個資料點都要有**歷史位置感**（百分位），讓使用者知道「這個數字在歷史上是高還是低」。

## 八、AI 的角色

AI 評論不是附加功能，是讓使用者**加速得到結論**的工具。

### AI 在三個層次發揮作用：

**市場層**（每日產生，穩定輸出）
- 輸入：breadthScore、advanceRatio、concentration、新高低家數、成交量比
- 輸出：2–3 句描述今日結構特徵，點出一件最值得注意的事
- 例：「今日市場廣度 72，高於近一年中位數，上漲家數廣泛但成交量僅達 20MA 的 0.9 倍，屬量縮上漲格局。半導體族群資金集中度明顯升高，值得觀察。」

**族群層**（進入族群頁時產生）
- 輸入：近 20 日 RS 趨勢、整體買賣超方向、廣度變化
- 輸出：1–2 句說明族群近期資金動能狀態
- 例：「本週相對強弱連續上升，外資法人持續買超，廣度維持 65% 以上，動能結構健康。」

**個股層**（進入個股後產生）
- 輸入：相對強弱趨勢、外資/投信/主力方向、大單集中度
- 輸出：1–2 句說明個股籌碼結構
- 例：「連續 3 日相對強弱為正，外資持續買超，大單集中度 78 偏高，短線籌碼集中。」

### AI 的設計原則：
- **穩定輸出**：不是 optional，每次進入對應層次都應有 AI 評語
- **只描述，不建議**：說明數據狀態，不給出「應該買」、「應該賣」等建議
- **失敗有替代**：若 AI 服務失敗，顯示最近一次的快取評語，而非空白

## 六、功能範圍

### 在範圍內（In Scope）

| 功能 | 說明 |
| ---- | ---- |
| 大盤市場結構總覽 | 廣度、資金、指數趨勢、新聞 |
| 族群熱力圖 + 多指標切換 | 漲跌/廣度/法人/強度 |
| 族群詳情頁 | RS 趨勢、資金走向、個股分類 |
| 個股結構表格 | 今日概覽 / 區間表現 / 籌碼風險 三 Tab |
| 個股籌碼頁 | 法人 + 主力 + 大單 + 融資融券 + K線 |
| 個股基本面概覽 | P/E、P/B、ROE、EPS 年增率、營收年增率（含歷史位階）|
| AI 評論（三層） | 市場 / 族群 / 個股 |

### 不在範圍內（Out of Scope）

| 功能 | 原因 |
| ---- | ---- |
| 即時盤中資料 | 定位是盤後研究，不是即時監控 |
| 投資建議 / 選股訊號 | 工具只提供資料，判斷由使用者自己做 |
| AI 預測（股價走向）| 超出工具定位，且不可靠 |
| 主力分點明細 | 資料複雜度高，先做更核心的功能 |
| 大盤委買委賣即時比 | 盤中指標，不符合盤後使用情境 |

## 七、設計原則

1. **每層資訊都要有歷史位置感**
   重要數字旁邊要顯示歷史百分位標籤（偏低 / 中位 / 偏高 / 極高），讓使用者不需要自己記歷史數據。

2. **快速掃描不需要跳頁**
   主頁應能在不切換頁面的情況下完成市場結構判斷。族群頁應能在不離開族群頁的情況下確認個股籌碼。

3. **研究路徑是 Top-down，但支援跳入**
   預設流程是大盤→族群→個股，但使用者應能從任何一層開始，不被強制線性閱讀。

4. **AI 是加速器，不是替代思考**
   AI 評語幫助使用者快速掌握資料摘要，但所有結論還是使用者自己做的。

5. **不做多空判斷，但要指出異常**
   工具不說「應該買」，但可以說「今天這個數字與過去相比偏高，值得注意」。

## 附錄：可用資料來源參考

以下是目前系統可取得的資料類型（作為功能開發的資料基礎確認用）：

- 三大法人買賣超（外資、投信、自營商）
- 主力分點資料
- 資券現沖資料（融資增減、融券增減、當沖比）
- 族群排行（法人、籌碼、價量、基本面各維度）
- 大盤廣度指標（上漲家數、創新高低家數、站上均線比例）
- 個股財務面（EPS、毛利率、ROE、本益比、股價淨值比）
- 個股區間漲幅
- 相關新聞
